Comandos básicos de git
Nombre del comando Atributo(ocional) Descripción
git init No tiene Es el comando que sirve para invocar la aparición de un nuevo repositorio.
Es la carpeta en la que están los archivos de tu proyecto.
Al abrir la terminal y ejecutar "git init", indicamos que esta es la carpeta central de los archivos.
  • Staging: Es el área en la memoria RAM donde se irán agregando los cambios temporalmente
  • Repository(repositorio/master) Es el directorio que conocemos como /.git/, lugar en el que se guardan todos los cambios en disco duro (la carpeta es un elemento oculto en el explorador de archivos)

git add >>Nombre del archivo de texto plano.extension<< Esta funcion almacena los cambios de un archivo determinado en el "staging" y lo prepara para almacenarlo en el repositorio. Si el nombre de archivo se sustituye por un punto (.), se guardarán todos los cambios hechos en los todos archivos
git rm --cached >>nombre del archivo<< Es el inverso de la funcion add, es decir, elimina un archivo determinado del staging
git commit -m ">mensaje o comentario acerca de la version entre comillas<" almacena el archivo en el repositorio como una nueva versión, transfiriendolo del staging al repository.
