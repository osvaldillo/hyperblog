Git init

Es el comando que sirve para invocar la aparición de un nuevo repositorio.
Es la carpeta en la que están los archivos de tu proyecto.

Al abrir la terminal y ejecutar "git init", indicamos que esta es la carpeta central de los archivos.

Cuando ejecutamos este comando, además, creamos:

  • Staging: Es el área en la memoria RAM donde se irán agregando los cambios temporalmente
  • Repository(repositorio/master) Es el directorio que conocemos como /.git/, lugar en el que se guardan todos los cambios en disco duro (la carpeta es un elemento oculto en el explorador de archivos)

git config --global

... user.name >>Nombre del creador de la versión<<: Es el comando para determinar el creador o de alguno de los editores dentro del repositorio

...user.email >>Correo electrónico del creador<<: Es el comando para determinar el Correo electrónico de alguno de los editores dentro del repositorio

git add

... >>Nombre del archivo de texto plano.extension<< Esta funcion almacena los cambios de un archivo determinado en el "staging" y lo prepara para almacenarlo en el repositorio. Si el nombre de archivo se sustituye por un punto (.), se guardarán todos los cambios hechos en los todos archivos

git rm --cached >>nombre del archivo<<

Es el inverso de la funcion add, es decir, elimina un archivo determinado del staging

git commit

...-m ">mensaje o comentario acerca de la version entre comillas<": almacena el archivo en el repositorio como una nueva versión, transfiriendolo del staging al repository.

git status

Nos muestra el estatus de nuestro proyecto, así como los cambios que necesitan ser actualizados

git log --stat

Nos muestra los cambios específicos de cada commit desde el más reciente hasta el más antiguo

git show

... >>archivo.extension<< Muestra los cambios hechos al archivo en el último commit

git dif

Muestra las diferencias entre los cambios que está en Stangig y los que aun no se han añadido

...#commit1 #commit2: Muestra la diferenciia entre dos versiones de los commits

git reset

... #commit --soft: devuelve la versión de un commit especificado sin borrar los demás commits

... #commit --hard: devuelve la versión de un commit especificado borrando los demás commits anteriores al commit especificado
